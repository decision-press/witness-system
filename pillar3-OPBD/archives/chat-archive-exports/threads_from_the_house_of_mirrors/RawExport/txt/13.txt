## 1. Get the TGD Astro project onto Linux

### Option A: You *already* have a GitHub repo for TGD (e.g. `decision-press`)

Use this if the Windows box is already pushing somewhere.

```bash
mkdir -p ~/Projects
cd ~/Projects

# clone the existing TGD repo
git clone git@github.com:YOUR-ACCOUNT/decision-press.git
# or https URL if you haven’t set SSH keys
# git clone https://github.com/YOUR-ACCOUNT/decision-press.git

cd decision-press

# install dependencies
npm install

# run local dev
npm run dev
```

Boom: TGD local on Linux.

### Option B: No remote yet, start fresh Astro + Git

```bash
mkdir -p ~/Projects
cd ~/Projects

mkdir tgd-site
cd tgd-site

# create Astro project in this directory
npm create astro@latest .

# when it asks:
# - project type: "empty" or "blog" or whatever you had before
# - typescript: yes
# - strict: your call
# - install deps: yes
# - init git: you can say no here since we’ll do it manually
```

Then:

```bash
npm install
npm run dev
```

If dev server runs, we’re alive.

Now initialize git:

```bash
git init

# set identity if git hasn’t been configured on this machine
git config user.name "Your Name"
git config user.email "you@example.com"

git add .
git commit -m "Initial TGD Astro setup on Linux"
```

If you want to hook it to a remote later:

```bash

# or some new repo
```

## 2. Create a home for the JSON / HTML exports inside TGD

Let’s treat the exports as an internal archive for future parsing, not something WordPress-style flung in random places.

From inside your TGD repo directory (whatever it’s called):

```bash
cd ~/Projects/decision-press   # or ~/Projects/tgd-site

mkdir -p witness-archive/chat-exports
```

So structure looks like:

```text
tgd-site-or-decision-press/
  src/
  public/
  astro.config.mjs
  package.json
  witness-archive/
    chat-exports/
```

Nice neat silo for “Jetson brain dumps.”

---

## 3. Put the JSON / HTML exports into the repo

Assuming you downloaded your ChatGPT export files to `~/Downloads` and they look something like:

- `chatgpt_conversations.json`
- `chatgpt_conversations.html`
- or some monstrous `my-data-2025-11-09.json`

Move or copy them:

```bash
cd ~/Projects/decision-press   # or ~/Projects/tgd-site

# copy all related exports in
cp ~/Downloads/*chat*export*.json witness-archive/chat-exports/ 2&gt;/dev/null || true
cp ~/Downloads/*chat*export*.html witness-archive/chat-exports/ 2&gt;/dev/null || true

# or just explicitly:
cp ~/Downloads/chatgpt_conversations.json witness-archive/chat-exports/
cp ~/Downloads/chatgpt_conversations.html witness-archive/chat-exports/
```

If you want to be a responsible future-you, add a tiny README in there:

```bash
cat &gt; witness-archive/chat-exports/README.md &lt;&lt; 'EOF'
# ChatGPT Export Archive

Raw JSON and HTML exports from ChatGPT, used as source material
for The God Decision / Witness System.

These are *inputs*, not rendered content. Do not edit by hand unless
you know exactly what you’re doing.
EOF
```

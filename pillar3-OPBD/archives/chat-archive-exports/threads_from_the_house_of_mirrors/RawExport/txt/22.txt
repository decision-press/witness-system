## ⚙️ Common Reasons Windows Doesn’t Detect the Fantom

### 1. **USB Driver Mode not set to “VENDOR”**
This is the *#1 cause*.

By default, Fantoms ship with the USB Driver set to **“GENERIC”** — which only exposes basic MIDI (class-compliant) functions and **blocks the Roland driver** from linking.

#### ✅ Fix:
On the Fantom:
1. Press **MENU**.  
2. Choose **System**.  
3. Scroll to **General → USB Driver**.  
4. Change from `GENERIC` → `VENDOR`.  
5. Press **Write** (so it saves that setting).  
6. Power cycle the Fantom.  

Then plug it back in via USB, and Windows should immediately show:  
&gt; “Installing device driver software” → “FANTOM (MIDI)” and “FANTOM (AUDIO)”  

---

### 2. **USB Port Power or Hub Interference**
Roland devices are sensitive to USB hubs, front-panel ports, or low-power laptop ports.

#### ✅ Fix:
- Plug the Fantom **directly into a rear motherboard USB port** on the Dell if possible.  
- Avoid docking stations or hubs during first install.  
- Use a short, shielded USB-B cable (printer-style).  

---

### 3. **Driver Installation Order**
If you plugged the Fantom in *before* installing the Roland driver, Windows might have cached a **generic USB Audio Device** entry.  
That prevents the Roland driver from binding later.

#### ✅ Fix:
1. Unplug the Fantom.  
2. Open **Device Manager → View → Show hidden devices.**  
3. Expand **Sound, video and game controllers** and **Universal Serial Bus controllers.**  
4. Delete (right-click → Uninstall) any entries called:
   - “USB Audio Device”
   - “FANTOM (Generic)”  
5. Reboot.  
6. Run the **Roland Fantom driver installer** again.  
7. When prompted, **only then** plug the Fantom in.

---

### 4. **Windows Security Policy Blocking Unsigned Drivers**
Roland’s driver *is* signed, but sometimes Windows SmartScreen or Defender intercepts it, especially if the system isn’t connected to the internet during install.

#### ✅ Fix:
- Temporarily disable SmartScreen or install in “Administrator” mode.  
- Reboot and check if `FANTOM` appears in Device Manager.  

---

### 5. **Wrong Cable Type**
The Fantom’s USB-B port requires a **data-capable cable** (some printer cables are charge-only).  
Try swapping the cable if it still won’t detect — this resolves more problems than you’d expect.

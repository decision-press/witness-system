### How Astro reads this

- The top block between `---` and `---` is called **frontmatter**.
  - This runs in Node at build time (server-side).
  - You can import components, define variables, fetch data, etc.
- Everything **below** the second `---` is the template:
  - Mostly HTML
  - With some Astro/JSX-like `{title}` interpolation
  - And component tags like `&lt;Counter client:load /&gt;`

You can think of it as:

&gt; Frontmatter = “prepare data &amp; imports”  
&gt; Template = “render the HTML + components using that data”

1. At build time, Astro renders a **plain `&lt;button&gt;`** in the HTML with initial text like `Count: 0`.
2. Because `index.astro` uses:
   ```astro
   &lt;Counter client:load /&gt;
   ```
   Astro also:
   - bundles this React component as JavaScript
   - sends that JS to the browser
   - **hydrates** the button (attaches the `onClick` listener and React state)

Result:  
The page **loads like static HTML**, but the button behaves like a fully interactive React component.

That’s the essence of the **island** pattern.
